## 2024-04-13 - PyTorch MultiheadAttention Optimization with need_weights=False
**Learning:** By default, PyTorch's `nn.MultiheadAttention` computes and returns attention weights (shape N x N), allocating extra memory and missing out on optimized attention backends (like FlashAttention) even when the user discards these weights (e.g. `attn_out, _ = self.attn(...)`). Setting `need_weights=False` avoids this. However, it's crucial to ONLY apply this optimization when the returned attention weights are discarded into `_` or not used. If the returned attention weight is explicitly unpacked into a named variable (e.g. `cross_attn_weights = ...`), setting `need_weights=False` returns `None` for the weights, which might break downstream logic or expected behavior.
**Action:** Always search for `nn.MultiheadAttention` calls. If the second return value (the weights) is assigned to `_`, add `need_weights=False` to the call to speed up computation and reduce memory footprint. If it's assigned to a named variable, avoid setting `need_weights=False` unless you rewrite the downstream logic.
