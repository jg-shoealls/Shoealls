## 2024-04-15 - Optimizing nn.MultiheadAttention with need_weights=False
**Learning:** PyTorch's `nn.MultiheadAttention` calculates and returns attention weights by default, which consumes memory and computation. Setting `need_weights=False` disables this and allows PyTorch to use optimized backends like FlashAttention. However, this codebase has an edge case in `src/models/reasoning_engine.py` where attention weights (`cross_attn_weights`) are explicitly unpacked from the return tuple. Using `need_weights=False` there would cause it to return `None` for the weights, which could cause hidden regressions or break expected downstream usage.
**Action:** When adding `need_weights=False` to `nn.MultiheadAttention` calls to optimize performance, always check the call site to ensure the returned attention weights are not being explicitly unpacked or used later in the code. Only apply it (like in `src/models/fusion.py`) when the weights are genuinely discarded (e.g., `attn_out, _ = ...`).
