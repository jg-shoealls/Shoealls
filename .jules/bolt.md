## 2024-05-24 - PyTorch MultiheadAttention Optimization
**Learning:** PyTorch's `nn.MultiheadAttention` computes attention weights by default, which incurs unnecessary memory allocation and computation if the weights are discarded (`_, _ = mha(...)`). Setting `need_weights=False` avoids this and enables optimized backends like FlashAttention. However, avoid applying this when the weights are unpacked into named variables (e.g., `out, cross_attn_weights = mha(...)`), as unpacking `None` might break expected downstream usage or cause hidden regressions.
**Action:** When using `nn.MultiheadAttention`, always set `need_weights=False` if the attention weights are not needed. Double check the variable assignment to ensure the returned weights aren't unpacked into a named variable that could be relied upon later.
