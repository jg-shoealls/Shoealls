## 2024-05-18 - PyTorch MultiheadAttention Optimization
**Learning:** By default, PyTorch's `nn.MultiheadAttention` computes and returns attention weights. In this codebase's architecture (`src/models/fusion.py`), the attention weights are explicitly discarded with `_`. However, `need_weights=False` should be avoided when the output is explicitly unpacked into a named variable (e.g., `cross_attn_weights` in `src/models/reasoning_engine.py`), as unpacking `None` causes hidden regressions or breaks expected downstream usage.
**Action:** When using `nn.MultiheadAttention` and the returned attention weights are discarded, pass `need_weights=False` to prevent unnecessary computation and memory allocation. This also enables optimized attention backends like FlashAttention. Always check if the output is unpacked into a named variable before applying.
