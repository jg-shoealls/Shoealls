## 2024-04-14 - PyTorch MultiheadAttention Memory and Compute Optimization
**Learning:** In this codebase, the PyTorch `nn.MultiheadAttention` module is heavily used for multimodal feature fusion. If the attention weights are unused, omitting the `need_weights=False` argument results in unnecessary computation and memory allocation to calculate and return the weight matrix. Setting `need_weights=False` prevents this and allows PyTorch to utilize highly optimized, memory-efficient attention backends like FlashAttention under the hood. However, this optimization should be skipped when the returned weight matrix is explicitly unpacked into a named variable (e.g., `cross_attn_weights` in `reasoning_engine.py`), because returning `None` instead of the weight matrix could crash downstream unpacking or indexing logic.
**Action:** When calling `nn.MultiheadAttention` where the attention weights (`_`) are discarded, always pass `need_weights=False` to leverage optimized backends like FlashAttention. Always verify that the second return value is indeed discarded or assigned to `_` before applying this optimization.
