## 2025-05-02 - Optimize MultiheadAttention by disabling unnecessary weight computation
**Learning:** In PyTorch, calling `nn.MultiheadAttention` with `need_weights=False` prevents unnecessary computation and memory allocation if attention weights are discarded, and also enables optimized attention backends (like FlashAttention). However, avoid applying it if the returned attention weights are explicitly unpacked into a named variable (e.g., `cross_attn_weights` in `src/models/reasoning_engine.py`), as unpacking `None` may cause hidden regressions or break expected downstream usage.
**Action:** When using `nn.MultiheadAttention` in PyTorch, identify calls where the weights return value is discarded (e.g., unpacked as `_`) and add `need_weights=False` to optimize execution, but always check downstream usage first.
