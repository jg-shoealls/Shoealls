## 2024-05-20 - Optimize MultiheadAttention Memory and Compute Usage
**Learning:** In PyTorch, `nn.MultiheadAttention` computes attention weights by default even if they are not used. Unpacking `_` doesn't stop the computation. However, when optimizing with `need_weights=False`, avoid applying it if the returned attention weights are explicitly unpacked into a named variable (e.g., `cross_attn_weights` like in `src/models/reasoning_engine.py` line 136), as unpacking `None` may cause hidden regressions or break expected downstream usage.
**Action:** When using `nn.MultiheadAttention`, if attention weights are unpacked to `_`, add `need_weights=False` to prevent unnecessary computation and enable optimized attention backends (like FlashAttention). If unpacked to a named variable, ensure `need_weights=True` (or omit, as it's default).
